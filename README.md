# Atividade de Designer Patters

O que é SOLID?

SOLID é um acrônimo para cinco princípios de design de software orientado a objetos, popularizados por Robert C. Martin (Uncle Bob).
 O objetivo desses princípios é criar sistemas mais fáceis de manter, entender e estender ao longo do tempo.

 Os cinco princípios são:

S - Single Responsibility Principle (Princípio da Responsabilidade Única)

O - Open/Closed Principle (Princípio Aberto/Fechado)

L - Liskov Substitution Principle (Princípio da Substituição de Liskov)

I - Interface Segregation Principle (Princípio da Segregação de Interface)

D - Dependency Inversion Principle (Princípio da Inversão de Dependência)

Exemplos usados nos arquivos:

- ocpExample.js: Exemplo do Princípio Aberto/Fechado (Open/Closed Principle - OCP)

Representado pela letra "O" em SOLID, o Princípio Aberto/Fechado estabelece que "entidades de software (classes, módulos, funções, etc.) devem estar abertas para extensão, mas fechadas para modificação". Isso significa que o comportamento de uma entidade pode ser estendido sem alterar seu código fonte existente, promovendo a reutilização e a manutenção do código.

- srpExample.js: Exemplo do Princípio da Responsabilidade Única (Single Responsibility Principle - SRP)

Representado pela letra "S" em SOLID, o Princípio da Responsabilidade Única estabelece que "uma classe deve ter apenas uma razão para mudar". Isso significa que cada classe deve ser responsável por uma única funcionalidade ou responsabilidade dentro do sistema, o que facilita a manutenção e a compreensão do código.
